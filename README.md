## 1. Название 🏷️
**Модуль согласования документов для системы электронного документооборота предприятия**

**Approval Module for Enterprise Document Management System**

---

## 2. Краткое описание 📝
Модуль предназначен для автоматизации процесса согласования документов на предприятии. Он реализует последовательную маршрутизацию документа между сотрудниками, контроль сроков рассмотрения, фиксацию решений и ведение полной истории согласования.

### Основные возможности ✨
- 🚀 Запуск процесса согласования с заданным списком сотрудников
- 🔄 Пошаговая передача документа между согласующими
- ✅ Принятие решений (согласовано / отклонено) с комментариями
- ⏰ Автоматический контроль просрочки выполнения
- 📚 Хранение полной истории согласования
- 📧 Информирование сотрудников о необходимости принять решение

## 4. Инструкция по установке 🛠️

### Шаг 1. Проверьте наличие .NET SDK ✅
Откройте терминал (командную строку) и выполните команду:

```bash
dotnet --version
```

Если .NET не установлен, скачайте и установите его с официального сайта:  
🔗 https://dotnet.microsoft.com/download

---

### Шаг 2. Создайте папку для проекта 📁

```bash
mkdir ApprovalModule
cd ApprovalModule
```

---

### Шаг 3. Создайте консольное приложение 🆕

```bash
dotnet new console -n ApprovalModule
cd ApprovalModule
```

---

### Шаг 4. Замените содержимое файла Program.cs 📄
Откройте файл `Program.cs` в любом текстовом редакторе и замените его содержимое на код, представленный в разделе **"Структура проекта"**.

---

### Шаг 5. Создайте остальные файлы проекта 📂
Создайте следующие файлы в папке проекта:
- `Models.cs` — модели данных 📦
- `ApprovalService.cs` — основная логика модуля ⚙️

Скопируйте в них соответствующий код из раздела **"Структура проекта"**.

---

### Шаг 6. Соберите проект 🔨

```bash
dotnet build
```

---

## 5. Инструкция по запуску 🚀

### Запуск в командной строке:

```bash
dotnet run
```

### Запуск в Visual Studio:
1. Откройте файл решения `ApprovalModule.csproj`
2. Нажмите клавишу `F5` или выберите меню **"Отладка"** → **"Запустить без отладки"**
